# 相遇点到入环点的距离关系（用你的比喻继续）

你用「无限长直线」抽象环的想法非常棒！让我们继续用这个比喻来理解相遇后的距离关系：

## 1. 相遇时的距离计算

假设：
- 从起点（链表头）到入环点的距离是：`a`步
- 环的长度是：`c`步
- 慢指针进环时，快指针已经在环内走了：`x`步（0 ≤ x < c）

因为快指针速度是慢指针的2倍，所以：
- 当慢指针走了`a`步到达入环点时，快指针已经走了`2a`步
- 快指针在环内的位置：`(2a - a) % c = a % c`步（也就是你说的`x = a % c`）

## 2. 为什么会追上？

现在，慢指针刚进入环，快指针在环内的位置是`x`步。由于环被抽象为无限长直线，相当于：
- 慢指针在位置`0`（环的入口）
- 快指针在位置`x`
- 慢指针速度：1步/次
- 快指针速度：2步/次

快指针需要追上慢指针，相对距离是`x`（因为快指针在后面），相对速度是`2-1=1`步/次，所以需要`x`次移动才能追上。

## 3. 相遇点的位置

追上时：
- 慢指针走了`x`步，位置在环内的`x`步处
- 快指针走了`2x`步，从初始位置`x`出发，最终位置是`x + 2x = 3x`，但由于环长是`c`，所以实际位置是`3x % c`

但因为`x = a % c`，所以`a = k*c + x`（k是整数），代入后：
- 慢指针在环内的位置：`x`步
- 快指针在环内的位置：`3x % c = (3*(a % c)) % c = (3a) % c`

不过更简单的方式是看总距离：
- 慢指针总距离：`a + x`
- 快指针总距离：`2*(a + x)`

## 4. 关键结论：相遇点到入环点的距离 = 起点到入环点的距离

根据上面的总距离，我们有：
`2*(a + x) = a + x + n*c`（n是快指针在环内走的圈数）

化简得：`a + x = n*c` → `a = n*c - x`

这意味着：
- `a`（起点到入环点）
- 等于 `n*c - x`（绕n圈减去x步）
- 而`c - x`就是相遇点到入环点的距离！

因为绕n圈相当于没动，所以`a = c - x`，也就是：
**起点到入环点的距离 = 相遇点到入环点的距离**

- 为什么相遇后把一个指针放回起点，两个指针同速前进会在入环点相遇？
