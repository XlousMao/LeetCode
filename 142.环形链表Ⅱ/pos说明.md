# 关于题目中`pos`参数的说明

## 1. `pos`是什么？

`pos`是**评测系统内部**用来构建测试用例的参数，**不是我们需要在代码中处理的输入**！

举个例子：
- 如果题目说`pos = 2`，评测系统会创建这样的链表：
  ```
  0 -> 1 -> 2 -> 3 -> 4
          ↑_________↓
  ```
  也就是链表尾节点（4）连接到索引为2的节点（节点值为2），形成环。

- 如果`pos = -1`，评测系统会创建普通的无环链表：
  ```
  0 -> 1 -> 2 -> 3 -> 4 -> null
  ```

## 2. 为什么我们不需要关心`pos`？

因为：
1. 评测系统不会把`pos`作为输入参数传给我们的函数
2. 我们的任务是通过链表的`head`指针，自己找出是否有环以及入环点
3. `pos`只是系统用来验证我们答案正确性的内部参数

## 3. 总结

- `pos`是系统用来构建测试链表的「幕后参数」
- 我们的代码完全不需要考虑这个参数的存在
- 只需要专注于：给定`head`指针，如何判断是否有环并找出入环点
