# 快慢指针找入环点的直观解释

## 1. 基本概念

假设链表结构如下：
- 从链表头到入环点的距离为：**a**（单位：步）
- 环的长度为：**c**（单位：步）
- 快慢指针相遇时，慢指针在环内走了：**b**（单位：步）

```
链表头 → ... → 入环点 → ... → 相遇点 → ... → 入环点 → ...
        a步         b步         (c - b)步
```

## 2. 为什么快慢指针会相遇？

- 慢指针速度：1步/次
- 快指针速度：2步/次

如果链表有环，快指针一定会追上慢指针，因为快指针每次比慢指针多走1步，相当于在以相对速度1步/次接近慢指针。

## 3. 相遇时的距离关系

当快慢指针第一次相遇时：
- 慢指针走的总距离：`a + b`
- 快指针走的总距离：`a + b + k*c`（k是快指针在环内走的圈数，k ≥ 1）

由于快指针速度是慢指针的2倍，所以：
`2*(a + b) = a + b + k*c`

简化后得到：
`a + b = k*c`

## 4. 为什么相遇后能找到入环点？

从上面的等式可以得出：
`a = k*c - b = (k-1)*c + (c - b)`

这意味着：
- 从链表头到入环点的距离（a）
- 等于从相遇点出发，绕环 (k-1) 圈后再走 (c - b) 步的距离
- 也就是从相遇点到入环点的距离（因为 (k-1) 圈刚好回到原来的位置）

所以：
1. 将一个指针放回链表头
2. 另一个指针留在相遇点
3. 两个指针以相同速度（1步/次）前进
4. 它们会在入环点相遇

## 5. 举个具体例子

假设：a=2（链表头到入环点2步），c=4（环长4步）

```
链表头 → 1 → 入环点 → 3 → 相遇点 → 5 → 入环点
        1步    2步    3步    4步    5步
```

- 慢指针走：a + b = 2 + 1 = 3步（到达相遇点）
- 快指针走：2*3 = 6步 = a + b + c = 2 + 1 + 3 = 6步
- 相遇后，慢指针从头开始走2步到入环点
- 快指针从相遇点走2步：相遇点→5→入环点
- 两人在入环点相遇
